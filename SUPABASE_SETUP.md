# Optional global leaderboard with Supabase

FlagCraft uses a local leaderboard until both values in `config.js` are filled. For a shared leaderboard, create a Supabase project and run the SQL below in its SQL editor.

```sql
create table if not exists public.flagcraft_scores (
  id bigint generated by default as identity primary key,
  player_id text not null check (char_length(player_id) between 8 and 96),
  username text not null check (
    char_length(username) between 2 and 16
    and username ~ '^[[:alnum:]_ -]+$'
  ),
  score integer not null check (score between 0 and 50000000),
  correct integer not null check (correct between 0 and 100000),
  total integer not null check (total between 0 and 100000 and correct <= total),
  duration integer not null check (duration between 0 and 86400),
  mode text not null check (mode in ('classic', 'survival', 'blitz')),
  difficulty text not null check (difficulty in ('easy', 'hard')),
  quiz_type text not null check (quiz_type in ('flags', 'capitals')),
  created_at timestamptz not null default now(),
  unique (player_id, mode, difficulty, quiz_type),
  check (score <= (correct * 700) + 700),
  check (mode <> 'classic' or (total <= 20 and correct <= 20 and duration <= 600)),
  check (mode <> 'blitz' or duration <= 120)
);

alter table public.flagcraft_scores enable row level security;

drop policy if exists "flagcraft public read" on public.flagcraft_scores;
create policy "flagcraft public read"
on public.flagcraft_scores for select
to anon
using (true);

drop policy if exists "flagcraft validated insert" on public.flagcraft_scores;
create policy "flagcraft validated insert"
on public.flagcraft_scores for insert
to anon
with check (
  mode in ('classic', 'survival', 'blitz')
  and difficulty in ('easy', 'hard')
  and quiz_type in ('flags', 'capitals')
  and correct between 0 and total
  and score <= (correct * 700) + 700
  and char_length(username) between 2 and 16
  and char_length(player_id) between 8 and 96
);

drop policy if exists "flagcraft validated update" on public.flagcraft_scores;
create policy "flagcraft validated update"
on public.flagcraft_scores for update
to anon
using (true)
with check (
  mode in ('classic', 'survival', 'blitz')
  and difficulty in ('easy', 'hard')
  and quiz_type in ('flags', 'capitals')
  and correct between 0 and total
  and score <= (correct * 700) + 700
  and char_length(username) between 2 and 16
  and char_length(player_id) between 8 and 96
);

create index if not exists flagcraft_board_rank_idx
on public.flagcraft_scores
(quiz_type, mode, difficulty, score desc, correct desc, duration asc);
```

Then copy the project URL and **anon/publishable** key into `config.js`:

```js
export const SUPABASE_URL = 'https://YOUR-PROJECT.supabase.co';
export const SUPABASE_ANON_KEY = 'YOUR-PUBLIC-ANON-KEY';
```

The anon/publishable key is intended for browser use when Row Level Security is enabled. Never place a service-role key in client-side code.

This setup validates record shape and plausible score bounds, but it is not a complete anti-cheat system. Since the game runs in the browser and anonymous users are not authenticated, a determined attacker can still forge plausible results. Strong competitive integrity requires a trusted backend that issues game sessions and verifies answers and timing server-side.
